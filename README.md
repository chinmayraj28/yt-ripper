## `> PREREQUISITES`

- **Node.js** 18+
- **yt-dlp** installed and available on `$PATH`

```bash
# macOS
brew install yt-dlp

# Linux
sudo curl -L https://github.com/yt-dlp/yt-dlp/releases/latest/download/yt-dlp -o /usr/local/bin/yt-dlp
sudo chmod a+rx /usr/local/bin/yt-dlp

# Windows (via pip)
pip install yt-dlp
```

### Download pipeline

```
yt-dlp -g [format]        ‚Üí  HLS stream URL(s)
         ‚Üď
ffmpeg [video URL]         ‚Üí  setStartTime / setDuration (trim)
       [audio URL]         ‚Üí  libmp3lame (MP3) or x264/AAC (MP4)
         ‚Üď
ReadableStream             ‚Üí  chunked Transfer-Encoding to browser
```

> **Why yt-dlp?** Both `ytdl-core` and `youtubei.js` fail to resolve playable URLs for current YouTube responses due to cipher and NSIG obfuscation. `yt-dlp` is actively maintained and handles these mitigations automatically.
